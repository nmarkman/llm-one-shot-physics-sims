LLM One-Shot Physics Sims

The same prompt, given once to different LLMs. No iteration, no follow-ups — just a single shot at building an interactive physics simulator. Compare the results.

About this project

Each model received the exact same prompt asking it to build an interactive physics simulator with bouncing balls inside a rotating polygon. Every result here is the raw, unedited output — one prompt, one response, no refinement.

It's a surprisingly revealing test. The prompt is complex enough to expose real differences in how models reason about physics, structure code, and design interfaces — but simple enough that every model can produce something functional.

What to look for

  • Physics accuracy — Do balls behave realistically? How is collision detection handled?
  • UI/UX quality — How polished are the controls? Is the interface intuitive?
  • Code sophistication — Mass-weighted physics? Wall friction? Re-constraining after collisions?
  • Visual design — Color choices, rendering effects, overall aesthetic

Adding a new model

Drop the model's output into sims/<model-name>/ with an index.html and sim.js, then add an entry to the MODELS array in this page's script.
